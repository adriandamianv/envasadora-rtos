La arquitectura, en tu idioma

Hexagonal de libro: el dominio no sabe de FreeRTOS, MySQL ni MQTT. Todo lo físico y lo externo son adaptadores.

— — las líneas punteadas son datos en movimiento: puertos ↔ adaptadores
[ 03 ]

El RTOS: cada adaptador es una tarea

FreeRTOS no es una capa — es el runtime. Prioridad ≠ jerarquía: la HMI "manda" en la arquitectura pero corre última; el determinismo vive en báscula → control → válvula.

tsk_control

evento · peso
tsk_bascula
sensor de peso
cada 50 ms
tsk_valvula
actuador
evento · comando
tsk_alarmas
vigila tolerancia y fallos
evento
tsk_mqtt
telemetría IoT
cada 500 ms
tsk_hmi
pantalla del operador
cada 200 ms

La barra ámbar = prioridad de scheduling · el LED parpadea al ritmo (escalado) de cada tarea · se sincronizan con colas y mutex, nunca variables globales a pelo.
